# Slider Component

Marwes provides a slider system with two layers:
**SliderField** for labeled form usage with edge values, description, and error text,
and the raw **Slider** atom for custom layouts.

## Component Architecture

```text
SliderField (Molecule)
Slider (Atom)
```

## Recommended: Purpose Components

Use purpose-built sliders when the control has a clear product meaning:

```vue
<script setup lang="ts">
import { BrightnessSlider, RadiusSlider, VolumeSlider } from "@marwes-ui/vue"
</script>

<template>
  <VolumeSlider description="Adjust the current output level for media playback." />
  <BrightnessSlider description="Tune the display intensity for the current workspace." />
  <RadiusSlider description="Control the corner rounding used across the component set." />
</template>
```

## Base: SliderField (Molecule)

Use `SliderField` for forms and settings panels where the control needs a visible label,
supporting text, and edge values.

```vue
<script setup lang="ts">
import { ref } from "vue"
import { SliderField } from "@marwes-ui/vue"

const radius = ref(24)
</script>

<template>
  <SliderField
    label="Radius"
    description="Controls the corner rounding used by the UI."
    v-model="radius"
    :slider="{
      min: 0,
      max: 48,
      step: 2,
      showTooltip: true,
    }"
  />
</template>
```

## Low-level Usage

Use `Slider` directly when you need a custom composition or when surrounding labels are already handled elsewhere.

```vue
<script setup lang="ts">
import { Slider } from "@marwes-ui/vue"
</script>

<template>
  <Slider :min="0" :max="100" :default-value="50" aria-label="Volume" show-touch-area />
</template>
```

## Supported Atom Features

- `min`, `max`, and `step` define the numeric range
- `value` and `onValueChange` support controlled usage
- `defaultValue` supports uncontrolled usage
- `showTooltip` displays the current numeric value above the thumb
- `showTouchArea` enables the larger touch halo treatment from the PDF spec
- `disabled` preserves the native disabled semantics
- width can be controlled with normal `style` or `class`, including full width layouts

## Accessibility Notes

- `Slider` renders a native `type="range"` control
- provide `ariaLabel` when there is no visible label nearby
- use `ariaLabelledBy` and `ariaDescribedBy` when the slider is connected to external text
- use `ariaValueText` when the raw numeric value needs a more descriptive spoken label

## Current Scope

`Slider` owns the interactive slider surface, tooltip, and touch-area visuals.
`SliderField` owns visible labels, description/error text, and min/max edge values.
Purpose wrappers encode common slider meanings on top of `SliderField`.

## Component Reference

**`VolumeSlider`** - Purpose wrapper for audio and media output levels

**`BrightnessSlider`** - Purpose wrapper for visual intensity adjustments

**`RadiusSlider`** - Purpose wrapper for radius and corner-rounding controls

**`SliderField`** - Recommended field wrapper with label, helper/error messaging,
and visible min/max values

**`Slider`** - Low-level horizontal range input with native keyboard behavior,
PDF-aligned hover/pressed/focus states, and optional tooltip and touch area visuals

## Story Coverage

- Atom: default, hover, pressed, focus, and disabled states
- Atom: tooltip and touch area variants
- Molecule: labeled field usage with description, error, and edge values
- Purpose: volume, brightness, and radius wrappers
- Full width layout usage
