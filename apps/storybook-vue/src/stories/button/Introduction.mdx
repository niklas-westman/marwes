# Button Components

Marwes provides a powerful button system with two complementary approaches:
**semantic components** for common patterns and **raw props** for advanced control.

Buttons are organized as Atom -> Variant wrappers -> Purpose wrappers to make Storybook navigation obvious.

## Button Architecture

```
Button (Atom)
     ↓
Variant Wrappers
  ├─ PrimaryButton
  ├─ SecondaryButton
  └─ TextButton
     ↓
Purpose Wrappers
  ├─ SubmitButton
  ├─ SaveButton
  ├─ CancelButton
  ├─ ConfirmButton
  ├─ VerifyButton
  ├─ CreateButton
  ├─ EditButton
  ├─ UploadButton
  ├─ DownloadButton
  ├─ CopyButton
  ├─ SearchButton
  ├─ FilterButton
  ├─ SortButton
  ├─ DropdownButton
  ├─ DangerButton
  ├─ LinkButton
  ├─ CloseButton
  └─ RefreshButton
```

## Atom: Button

Use the base `Button` atom when you need full control over `action`, `variant`, and rendering mode (`button` vs `a`):

```vue
<script setup lang="ts">
import { Button } from "@marwes-ui/vue"
</script>

<template>
  <Button variant="primary" action="button">Base Button</Button>
  <Button variant="secondary" action="navigate" href="/dashboard">Go to dashboard</Button>
</template>
```

## Recommended: Semantic Components

Use purpose-built components that automatically set AI-friendly metadata:

```vue
<script setup lang="ts">
import {
  CancelButton,
  SubmitButton,
  SaveButton,
  ConfirmButton,
  VerifyButton,
  CreateButton,
  EditButton,
  UploadButton,
  DownloadButton,
  CopyButton,
  SearchButton,
  FilterButton,
  SortButton,
  DropdownButton,
  DangerButton,
  LinkButton,
  CloseButton,
  RefreshButton,
} from "@marwes-ui/vue"
</script>

<template>
  <!-- ✅ Preferred - auto-sets action="cancel", variant="secondary" -->
  <CancelButton @click="handleCancel">Cancel</CancelButton>

  <!-- ✅ Preferred - auto-sets action="submit", uses form submission -->
  <SubmitButton>Save Changes</SubmitButton>

  <!-- ✅ Preferred - save-specific wrapper with icon and save metadata -->
  <SaveButton @click="handleSave">Save Draft</SaveButton>

  <!-- ✅ Preferred - affirmative success wrapper for confirmation flows -->
  <ConfirmButton @click="handleConfirm">Confirm</ConfirmButton>

  <!-- ✅ Preferred - success wrapper for verification/approval flows -->
  <VerifyButton @click="handleVerify">Verify Account</VerifyButton>

  <!-- ✅ Preferred - auto-sets action="create" -->
  <CreateButton @click="handleCreate">New Project</CreateButton>

  <!-- ✅ Preferred - edit-specific wrapper with icon and edit metadata -->
  <EditButton @click="handleEdit">Edit Profile</EditButton>

  <!-- ✅ Preferred - utility wrappers keep secondary styling with purpose metadata -->
  <UploadButton @click="handleUpload">Upload File</UploadButton>
  <DownloadButton @click="handleDownload">Download CSV</DownloadButton>
  <CopyButton @click="handleCopy">Copy Link</CopyButton>
  <SearchButton @click="handleSearch">Search</SearchButton>
  <FilterButton @click="openFilters">Filter</FilterButton>
  <SortButton @click="toggleSort">Sort</SortButton>
  <DropdownButton>More Actions</DropdownButton>

  <!-- ✅ Preferred - auto-sets action="delete", data-destructive="true" -->
  <DangerButton @click="handleDelete">Delete</DangerButton>

  <!-- ✅ Preferred - auto-sets action="navigate", renders as anchor -->
  <LinkButton href="/dashboard">Go to Dashboard</LinkButton>

  <!-- ✅ Preferred - close and refresh semantics without manual prop wiring -->
  <CloseButton @click="handleClose">Close</CloseButton>
  <RefreshButton @click="handleRefresh">Refresh</RefreshButton>
</template>
```

> **Skipped in the current docs surface:** `BackButton`, `NextButton`, `MessageButton`, `ShareButton`, `LoginButton`, `LogoutButton`, `HelpButton`, and `SettingsButton` remain intentionally omitted until that final purpose-button wave ships.

## Advanced: Raw Props

For dynamic or custom scenarios, use the base `Button` with explicit props:

```vue
<script setup lang="ts">
import { ref } from "vue"
import { Button, ButtonVariant, ButtonAction, ButtonSize } from "@marwes-ui/vue"

const isEditing = ref(false)
</script>

<template>
  <!-- Dynamic action based on state -->
  <Button
    :variant="ButtonVariant.primary"
    :size="ButtonSize.md"
    :action="isEditing ? ButtonAction.edit : ButtonAction.create"
  >
    {{ isEditing ? "Update" : "Create" }}
  </Button>

  <!-- Custom action not covered by semantic components -->
  <Button
    :variant="ButtonVariant.secondary"
    :action="ButtonAction.reset"
  >
    Reset Form
  </Button>
</template>
```

## Variant vs Action

### `variant` - Visual Style

Controls how the button **looks**. Choose based on visual hierarchy:

<ul>
  {buttonVariantVisualStyles.map((item) => (

      <strong>
        <code>{item.label}</code>
      </strong>{" "}
      - {item.text}
    </li>
  ))}
</ul>

### `action` - Semantic Purpose

Controls what the button **does** (AI-readable metadata):

<ul>
  {buttonActionSemanticPurposes.map((item) => (
    <li key={item.label}>
      <strong>
        <code>{item.label}</code>
      </strong>{" "}
      - {item.text}
    </li>
  ))}
</ul>

> **Note:** These are independent. A `DangerButton` uses `variant="primary"` (filled) with error styling, while `CancelButton` uses `variant="secondary"` (outlined).

## Component Reference

### Variant Buttons

**`PrimaryButton`** - Common primary style wrapper  
Sets: `variant="primary"`

**`SecondaryButton`** - Common secondary style wrapper  
Sets: `variant="secondary"`

**`TextButton`** - Common low-emphasis style wrapper  
Sets: `variant="text"`

### Purpose Buttons (AI-Friendly)

**`SubmitButton`** - Form submissions  
Sets: `action="submit"`

**`SaveButton`** - Save/persist actions  
Sets: `action="submit"`, default save icon

**`CancelButton`** - Cancel/dismiss actions  
Sets: `action="cancel"`, `variant="secondary"`

**`ConfirmButton`** - Affirmative confirmation actions  
Sets: `variant="success"`, positive outcome metadata

**`VerifyButton`** - Verification/approval actions  
Sets: `variant="success"`, verification metadata

**`CreateButton`** - Creating new items  
Sets: `action="create"`

**`EditButton`** - Edit/modify actions  
Sets: `action="edit"`, `variant="secondary"`

**`UploadButton`** - Upload/import actions  
Sets: `variant="secondary"`, upload metadata, upload icon

**`DownloadButton`** - Download/export actions  
Sets: `variant="secondary"`, download metadata, download icon

**`CopyButton`** - Copy/duplicate actions  
Sets: `variant="secondary"`, copy metadata, copy icon

**`SearchButton`** - Explicit search triggers  
Sets: `variant="secondary"`, search metadata, search icon

**`FilterButton`** - Filter and refine controls  
Sets: `variant="secondary"`, filter metadata, filter icon

**`SortButton`** - Sort/order controls  
Sets: `variant="secondary"`, sort metadata, ordering icon

**`DropdownButton`** - Disclosure or action-menu triggers  
Sets: `variant="secondary"`, dropdown metadata, chevron icon

**`DangerButton`** - Destructive actions  
Sets: `action="delete"`, error styling, `data-destructive="true"`

**`LinkButton`** - Navigation links  
Sets: `action="navigate"`, renders as `<a>` element

**`CloseButton`** - Close/dismiss affordances  
Sets: `action="cancel"`, `variant="secondary"`

**`RefreshButton`** - Refresh/reload actions  
Sets: `variant="secondary"`, refresh metadata

**Skipped until implemented** - `BackButton`, `NextButton`, `MessageButton`, `ShareButton`, `LoginButton`, `LogoutButton`, `HelpButton`, `SettingsButton`
