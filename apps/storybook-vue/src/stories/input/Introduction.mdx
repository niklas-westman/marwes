# Input Components

Marwes provides purpose-specific field components that encapsulate best practices
and automatically set AI-friendly metadata. Each field variant is optimized for
its specific use case with appropriate semantic HTML attributes.

Input domain groups everything related to text-like data entry in one place,
using a clear Atom -> Molecule -> Purpose progression.

### Field naming

Components that wrap the raw `Input` atom with label, helper text, and error semantics
are called **Fields** (e.g. `InputField`, `SelectField`, `TextareaField`, `RichTextField`, `DropdownField`, `SearchField`, `EmailField`).
The suffix signals that the component provides full field-level UX — accessible labelling,
validation display, and AI-friendly `data-purpose` attributes — so you don't have to wire
those concerns yourself.

```

  ├─ Input
  ├─ Select

Molecules
  ├─ InputField
```

## Atom: Input

Use `Input` directly when you need full layout control without label/helper/error wrappers:

```vue
<script setup lang="ts">
import { ref } from "vue"
import { Input } from "@marwes-ui/vue"

const value = ref("")
</script>

<template>
  <Input
    type="text"
    placeholder="Type here"
    v-model="value"
  />
</template>
```

## Atom: Select

Use `Select` for the compact select atom. Set `native` to `true` when you explicitly want the browser's built-in dropdown chrome:

```vue
<script setup lang="ts">
import { ref } from "vue"
import { Select } from "@marwes-ui/vue"

const country = ref("")
</script>

<template>
  <Select
    placeholder="Choose a country"
    :options="[
      { value: 'se', label: 'Sweden' },
      { value: 'us', label: 'United States' }
    ]"
    v-model="country"
  />
</template>
```

## Atom: Textarea

Use `Textarea` when you need multiline freeform text entry without field-level wrapper UI:

```vue
<script setup lang="ts">
import { ref } from "vue"
import { Textarea } from "@marwes-ui/vue"

const value = ref("")
</script>

<template>
  <Textarea
    rows="4"
    placeholder="Add more detail"
    v-model="value"
  />
</template>
```

## Atom: RichText

Use `RichText` when the authored content itself needs inline emphasis such as bold, italic, or underline. Use `Textarea` when the content should stay plain text.

```vue
<script setup lang="ts">
import { ref } from "vue"
import { RichText } from "@marwes-ui/vue"

const html = ref("")
</script>

<template>
  <RichText
    placeholder="Write a formatted update"
    v-model="html"
  />
</template>
```

## Recommended: Purpose Input Components

Use purpose-built field components that automatically set the correct `type`, `inputMode`, `autoComplete`, and other semantic attributes:

```vue
<script setup lang="ts">
import { ref } from "vue"
import {
  DropdownField,
  SearchField,
  PasswordField,
  EmailField,
  DateOfBirthField,
  ZipCodeField,
  PhoneField,
  URLField,
  CurrencyField,
} from "@marwes-ui/vue"

const query = ref("")
const country = ref("")
const password = ref("")
const email = ref("")
const dateOfBirth = ref("")
const zipCode = ref("")
const phone = ref("")
const url = ref("")
const amount = ref("")
</script>

<template>
  <DropdownField
    label="Country"
    v-model="country"
    :select="{
      placeholder: 'Choose a country',
      options: [
        { value: 'se', label: 'Sweden' },
        { value: 'us', label: 'United States' }
      ]
    }"
  />

  <!-- ✅ Search with automatic clear button -->
  <SearchField label="Search products" v-model="query" />

  <!-- ✅ Password with show/hide toggle -->
  <PasswordField label="Password" v-model="password" />

  <!-- ✅ Email with proper validation and keyboard -->
  <EmailField label="Email address" v-model="email" />

  <!-- ✅ Date of birth with native date semantics and birthday autofill -->
  <DateOfBirthField label="Date of birth" v-model="dateOfBirth" />

  <!-- ✅ ZIP/postal code with numeric keypad and autofill -->
  <ZipCodeField label="ZIP code" v-model="zipCode" />

  <!-- ✅ Phone with phone keyboard on mobile -->
  <PhoneField label="Phone number" v-model="phone" />

  <!-- ✅ URL with URL keyboard and validation -->
  <URLField label="Website" v-model="url" />

  <!-- ✅ Currency with decimal keyboard -->
  <CurrencyField label="Amount" currency="USD" v-model="amount" />
</template>
```

## When to Use Each Field

### DropdownField
- Figma-aligned dropdown presentation
- Cases where the custom Marwes open-state should stay visually consistent
- Forms that need the same field semantics as `SelectField`, but a fixed visual treatment

**Example:**
```vue
<script setup lang="ts">
import { ref } from "vue"
import { DropdownField } from "@marwes-ui/vue"

const country = ref("")
</script>

<template>
  <DropdownField
    label="Country"
    helperText="Uses the Marwes dropdown presentation."
    v-model="country"
    :select="{
      options: [
        { value: 'se', label: 'Sweden' },
        { value: 'us', label: 'United States' }
      ]
    }"
  />
</template>
```

### SearchField
- Search bars and filters
- Product search
- User/content lookup
- Any search functionality

**Example:**

```vue
</template>
```

## Molecule: InputField

Use the base `InputField` when you want built-in label, helper text, and error wiring:

```vue
<script setup lang="ts">
import { ref } from "vue"
import { InputField } from "@marwes-ui/vue"

const value = ref("")
</script>

<template>
  <InputField
    label="Custom field"
    v-model="value"
    :input="{
      type: 'text',
      inputMode: 'numeric',
      pattern: '[0-9]*'
    }"
  />
</template>
```

Use `RichTextField` when the editor needs the same label, helper text, and error treatment as other Marwes field wrappers:

## Common Props

Input-based field components support the same base props:

```ts
type BaseFieldProps = {
  label: string                    // Required: Field label
  helperText?: string             // Optional: Helper text below input
  error?: string                  // Optional: Error message (marks invalid)
  id?: string                     // Optional: Custom ID (auto-generated if omitted)
  ariaDescribedBy?: string        // Optional: Additional aria-describedby IDs
  modelValue?: string             // Vue v-model value
  input?: {
    value?: string                // Controlled value (React-parity API)
    defaultValue?: string         // Uncontrolled default value
    onValueChange?: (value: string) => void
    placeholder?: string
    disabled?: boolean
    readOnly?: boolean
    required?: boolean
    // ... and other input-specific props
  }
}
```

`SelectField` uses the same outer props (`label`, `helperText`, `error`, `id`, `ariaDescribedBy`) but forwards select control props under `select`. The custom Marwes dropdown state from Figma node `1364:7701` is the default. Use `select.native={true}` to opt into browser select chrome. `TextareaField` does the same for multiline control props under `textarea`.

`DropdownField` reuses the `SelectField` API, defaults `select.native` to `false`, and acts as the purpose-level wrapper for the custom Marwes dropdown presentation. Pass `select.native={true}` only when you intentionally want browser chrome instead.

`TextareaField` uses the same outer props (`label`, `helperText`, `error`, `id`, `ariaDescribedBy`) but forwards multiline control props under `textarea` instead of `input`.

## Controlled vs Uncontrolled

### Controlled (Recommended)
```vue
<script setup lang="ts">
import { ref } from "vue"
import { EmailField } from "@marwes-ui/vue"

const email = ref("")
</script>

<template>
  <EmailField label="Email" v-model="email" />
</template>
```

### Uncontrolled
```vue
<template>
  <EmailField
    label="Email"
    :input="{ defaultValue: 'user@example.com' }"
  />
</template>
```

## Validation Pattern

```vue
<script setup lang="ts">
import { computed, ref } from "vue"
import { EmailField } from "@marwes-ui/vue"

const email = ref("")
const error = computed(() => {
  if (!email.value) return "Email is required"
  if (!/^[^\\s@]+@[^\\s@]+\\.[^\\s@]+$/.test(email.value)) {
    return "Please enter a valid email address"
  }
  return undefined
})
</script>

<template>
  <EmailField
    label="Email address"
    :error="error"
    v-model="email"
  />
</template>
```

## AI-Friendly Attributes

All field variants automatically set `data-purpose` attributes for AI parsing:

```html
<div data-purpose="email">
  <label>Email address</label>
  <input type="email" inputmode="email" autocomplete="email" />
</div>

<div data-purpose="password">
  <label>Password</label>
  <input type="password" autocomplete="current-password" />
  <button aria-label="Show password">...</button>
</div>

<div data-purpose="currency" data-currency="USD">
  <label>Amount</label>
  <input type="text" inputmode="decimal" />
  <p>Amount in USD</p>
</div>
```

These attributes help AI tools understand field purpose and provide better assistance.
