# Divider Component

Marwes provides Divider as a single atom for separating related content without
introducing extra structure. Use it to create visual breaks between stacked
sections or as a vertical separator in inline layouts.

## Component Architecture

```
Divider (Atom)
```

## Recommended Usage

Use the raw `Divider` atom directly. Choose horizontal dividers for stacked
content and vertical dividers inside flex or inline layouts.

```vue
<script setup lang="ts">
import { Divider } from "@marwes-ui/vue"
</script>

<template>
  <Divider size="sm" />
  <Divider orientation="vertical" size="xs" />
</template>
```

## Usage Notes

- `size` controls both divider thickness and the built-in surrounding spacing
- `orientation="horizontal"` is the default for content sections
- `orientation="vertical"` works best in containers that define their own height

## Accessibility notes

**Divider is a real semantic separator.** `Divider` renders an `<hr>` with separator semantics.
Use it when the UI benefits from an actual content boundary, not just a visual line.

**Do not use Divider as a spacing helper.** If the layout only needs whitespace, use spacing
or container layout instead of inserting a semantic separator that assistive technology will read.

```vue
<!-- ❌ Adds separator semantics when the layout only needs space -->
<template>
  <Paragraph>Account name</Paragraph>
  <Divider />
  <Paragraph>Plan name</Paragraph>
</template>

<!-- ✅ Use layout spacing when there is no real section break -->
<template>
  <div style="display: grid; gap: 1rem;">
    <Paragraph>Account name</Paragraph>
    <Paragraph>Plan name</Paragraph>
  </div>
</template>
```

**Vertical dividers depend on container layout.** The divider exposes
`aria-orientation="vertical"`, but it still needs a parent layout that provides usable height and
alignment. Without that, the rule may be technically correct but visually unhelpful.

**Divider is unlabeled.** If the content needs a named section break such as `"Billing"` or `"Or"`,
use headings or dedicated text content rather than expecting `Divider` to carry that meaning.
