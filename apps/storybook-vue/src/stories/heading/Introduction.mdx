# Heading Component

Marwes provides `H1`, `H2`, and `H3` as atom-level typography primitives for
semantic structure and visual hierarchy. Choose the heading level by document
outline first, then use `size` only when the visual scale needs to differ from
the semantic level.

## Component Architecture

```
H1 / H2 / H3 (Atoms)
```

## Recommended Usage

Use the raw heading atoms directly to express page, section, and subsection
hierarchy.

```vue
<script setup lang="ts">
import { H1, H2, H3 } from "@marwes-ui/vue"
</script>

<template>
  <H1>Page Title</H1>
  <H2>Section Heading</H2>
  <H3>Subsection Heading</H3>
</template>
```

## Usage Notes

- Prefer `H1` for the page-level title, then nest `H2` and `H3` by content
  structure
- Use `size` when semantic level and visual scale need to differ
- Add `id` when headings should support anchor links or table-of-contents jumps

## Accessibility notes

**Choose the heading level by document outline first.** `H1`, `H2`, and `H3` render real
native heading elements. Screen readers use those levels to navigate the page structure.
Pick the level that matches the content hierarchy, not the font size you want.

**`size` changes the visual scale, not the semantic level.** An `H3` with `size="h1"`
is still an `<h3>` for assistive technology. Use this escape hatch sparingly when visual
hierarchy and document outline genuinely need to diverge.

```vue
<!-- ❌ Visual styling used instead of honest outline -->
<H3 size="h1">Billing</H3>

<!-- ✅ Semantic level stays honest; visual override only when truly needed -->
<H2 size="h1">Billing</H2>
```

**Do not use headings for decorative emphasis only.** If text needs to look larger or bolder
but does not introduce a real section, use another typography surface rather than inserting a
fake heading into the document outline.

**`ariaLabel` is rare for headings.** In most cases the visible heading text should be the
accessible name. Only add `ariaLabel` when product code has a real naming reason beyond the
visible text.

## Component Reference

**`H1`** - Top-level heading atom for page titles and primary content entry
points

**`H2`** - Section heading atom for major content groupings

**`H3`** - Subsection heading atom for nested content within a section
