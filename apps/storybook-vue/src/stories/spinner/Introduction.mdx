# Spinner Component

Marwes provides `Spinner` as a low-level loading atom for inline button states,
centered empty states, and standalone async feedback.

## Component Architecture

```
Spinner (Atom)
ButtonSpinner (Purpose)
EmptyStateSpinner (Purpose)
```

## Recommended Usage

Use the raw `Spinner` atom when product code needs direct control over visual
style, size, or accessibility behavior. Prefer `ButtonSpinner` and
`EmptyStateSpinner` when the loading context already matches those purpose
wrappers.

Pick the `variant` that matches the synced Figma showcase, choose a token `size`
for common cases, and keep the spinner `decorative` when nearby text already says
what is loading.

```vue
<script setup lang="ts">
import { Spinner } from "@marwes-ui/vue"
</script>

<template>
  <Spinner variant="classic" size="xs" decorative />
  <Spinner variant="dots-round" size="lg" ariaLabel="Loading account data" />
</template>
```

## Usage Notes

- `variant` maps to the seven synced Spinner styles from Figma
- `size` supports spinner scale tokens (`xs`/`sm`/`md`/`lg`) and explicit pixel numbers in code
- `decorative` should stay enabled when visible text or surrounding UI already communicates loading
- `ariaLabel` should be provided when the spinner must announce loading on its own
- CSS custom properties such as `--mw-spinner-indicator-color` and `--mw-spinner-track-color` can be overridden for contextual treatments

## Accessibility notes

**Decorative fallback:** A spinner with `decorative={false}` (`:decorative="false"`) but no
`ariaLabel` still hides from assistive technology — there is no fallback announced text without
a label. Always provide `ariaLabel` alongside `decorative={false}` when the spinner must
announce loading status on its own. Marwes emits a development warning when this combination
is detected.

**Reduced-motion:** The spinner animation slows from 800ms to 1600ms under
`prefers-reduced-motion: reduce` rather than stopping entirely. This is an intentional
design choice that keeps visible loading feedback present. Product teams should pair
spinners in reduced-motion contexts with sufficient visible loading text when waits
may be longer than a few seconds.

**Button loading:** The `Button` loading prop integrates through `ButtonSpinner` in both
adapters. The button stays `aria-busy` and disabled while loading; the spinner itself
stays decorative because the button label already communicates what is loading.

**`ButtonSpinner`** - Purpose wrapper for inline button loading treatments

**`EmptyStateSpinner`** - Purpose wrapper for centered empty-state loading treatments
