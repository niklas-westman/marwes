# Card Components

Marwes provides cards at two layers: **purpose components** for common semantic
surfaces and the raw **Card** atom for custom composition.

## Component Architecture

```
Purpose Card
    ↓
Card (Atom)
```

## Recommended: Purpose Components

Use purpose-built cards when the surface has a clear product meaning:

```vue
<script setup lang="ts">
import { ProductCard, ProfileCard, StatCard } from "@marwes-ui/vue"
</script>

<template>
  <ProductCard>
    <template #title>Pro plan</template>
    Unlimited projects, advanced analytics, and priority support.
  </ProductCard>

  <ProfileCard>
    <template #title>Niklas Westman</template>
    Design systems engineer with ownership of component primitives.
  </ProfileCard>

  <StatCard>
    <template #title>Monthly recurring revenue</template>
    $128,400
  </StatCard>
</template>
```

## Base: Card (Atom)

Use the raw `Card` atom when you need a custom surface layout:

```vue
<script setup lang="ts">
import { Card } from "@marwes-ui/vue"
</script>

<template>
  <Card :data-attributes="{ 'data-surface': 'summary-card' }">
    <template #title>Project status</template>
    Build pipeline is healthy.
  </Card>
</template>
```

### Card Features

- **Optional title header** - renders consistent header spacing when a `title` slot is provided
- **Composition-friendly body** - accepts any Vue content through the default slot
- **Purpose-ready wrapper** - supports `dataAttributes` so semantic variants can layer metadata

## Component Reference

### Purpose Components

**`ProductCard`** - Product summaries and offer surfaces

**`ProfileCard`** - Identity and account summary surfaces

**`StatCard`** - KPI and snapshot metric surfaces

### Atom

**`Card`** - Low-level titled surface container

- `Card (Atom)`
- `ProductCard`
- `ProfileCard`
- `StatCard`
