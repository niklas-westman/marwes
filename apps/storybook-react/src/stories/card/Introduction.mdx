# Card Components

Marwes provides cards at two layers: **purpose components** for common semantic
surfaces and the raw **Card** atom for custom composition.

## Component Architecture

```
Purpose Card
    ↓
Card (Atom)
```

## Recommended: Purpose Components

Use purpose-built cards when the surface has a clear product meaning:

```tsx
import { ProductCard, ProfileCard, StatCard } from "@marwes-ui/react"

<ProductCard title="Pro plan">
  Unlimited projects, advanced analytics, and priority support.
</ProductCard>

<ProfileCard title="Alex Morgan">
  Design systems engineer with ownership of component primitives.
</ProfileCard>

<StatCard title="Monthly recurring revenue">
  $128,400
</StatCard>
```

## Base: Card (Atom)

Use the raw `Card` atom when you need a custom surface layout:

```tsx
import { Card } from "@marwes-ui/react"

<Card
  title="Project status"
  dataAttributes={{ "data-surface": "summary-card" }}
>
  Build pipeline is healthy.
</Card>
```

### Card Features

- **Optional title header** - renders consistent header spacing when `title` is provided
- **Composition-friendly body** - accepts any React content as children
- **Purpose-ready wrapper** - supports `dataAttributes` so semantic variants can layer metadata

## Accessibility notes

**Card is a passive `<div>`.** The card shell has no implicit role, heading, or interactive
semantics. Screen readers see it as a generic container. Meaning comes entirely from the
content placed inside.

**Card title is a styled `<span>`, not a heading.** Passing a string as `title` renders
`<span class="mw-card__title">`. It does not update the document heading outline. If a
card surface needs to be findable by heading navigation, provide a real heading element
in the card body rather than relying on the `title` prop.

**Interactive card pattern.** If a card must be clickable, wrap its content in an `<a>` or
`<button>` rather than adding `onClick` directly to the `Card` surface. A plain `<div>`
with a click handler is unreachable by keyboard users and invisible to assistive technology
as an interactive control.

```tsx
// ❌ Click handler on a passive div — keyboard inaccessible
<Card onClick={handleClick}>Project status</Card>

// ✅ Anchor wraps the card content — keyboard and AT accessible
<a href="/projects/123">
  <Card>Project status</Card>
</a>
```

**`aria-*` and native div props pass through.** You can apply `aria-label`, `aria-labelledby`,
`role`, or other semantics directly to `Card` when product code needs to express richer meaning
for a specific composition.
