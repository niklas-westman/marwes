# Avatar Components

Marwes provides avatars at three layers: **purpose components** for common
identity contexts, **molecules** for presence and grouped layouts, and the raw
**Avatar** atom for custom composition.

## Component Architecture

```
Purpose Components
  ↓
AvatarBadge / AvatarGroup (Molecules)
  ↓
Avatar (Atom)
```

## Recommended: Purpose Components

Use semantic wrappers when the avatar surface has a clear product meaning.
They add AI-friendly `data-purpose` metadata while preserving the same visual
contracts.

```tsx
import { ProfileAvatar, PresenceAvatar, TeamAvatarGroup } from "@marwes-ui/react"

<ProfileAvatar initials="MW" />
<PresenceAvatar initials="MW" statusLabel="Online" />
<TeamAvatarGroup
  ariaLabel="Project members"
  items={[
    { id: "mw", initials: "MW" },
    { id: "nk", initials: "NK" },
    { id: "guest", type: "icon", ariaLabel: "Guest member" },
  ]}
  overflowCount={3}
/>
```

## Base Layers

Use the raw `Avatar` atom for standalone identity rendering. Use the molecules
when you need the exact presence and stacked-group patterns from Figma.

```tsx
import { Avatar, AvatarBadge, AvatarGroup } from "@marwes-ui/react"

<Avatar initials="MW" />
<AvatarBadge initials="MW" />
<AvatarGroup
  ariaLabel="Project members"
  items={[
    { id: "mw", initials: "MW" },
    { id: "nk", initials: "NK" },
    { id: "guest", type: "icon", ariaLabel: "Guest member" },
  ]}
  overflowCount={3}
/>
```

## Usage Notes

- `Avatar` supports `small`, `medium`, and `large`
- `Avatar` supports `initials`, `icon`, and `image`
- `AvatarBadge` adds the online green-dot indicator from Figma
- `AvatarGroup` renders the medium stacked layout with an optional overflow counter
- `ProfileAvatar` sets `data-purpose="profile"`
- `PresenceAvatar` sets `data-purpose="presence"`
- `TeamAvatarGroup` sets `data-purpose="team"`
- `initials` are normalized to two uppercase characters
- `alt` labels image avatars; `ariaLabel` also works as an alt-text fallback for image avatars
- non-image avatars use `ariaLabel`, or normalized initials, or fall back to `"Avatar"`
- `AvatarGroup` defaults the group label to `"Avatar group"` when no `ariaLabel` is provided
- `decorative` hides Avatar and AvatarBadge from assistive technology when nearby text
  already identifies the person

## Accessibility notes

**Image avatars need a name.** An image avatar rendered without `alt` and without `ariaLabel`
produces `<img alt="">`, which silently hides the person from assistive technology. Always
provide `alt` (preferred) or `ariaLabel` when the image communicates a person's identity.
Marwes emits a development warning when an image avatar has neither.

**Group label.** `AvatarGroup` uses a `<fieldset>` as its semantic group wrapper. Always
provide `ariaLabel` that describes the group in context, such as `"Project members"` or
`"Online teammates"`. The default `"Avatar group"` label is a safe fallback but rarely
context-rich enough for real product layouts.

**Overflow counter.** The overflow counter exposes a `role="img"` element with a label
such as `"3 more people"`. Provide a custom `overflowLabel` when the default wording does
not fit the product context.

**Decorative usage.** Use `decorative` only when nearby text already identifies the person
and the avatar is purely visual reinforcement. Do not use `decorative` on an identity surface
that carries information the user would not otherwise have.

## Component Reference

### Purpose Components

**`ProfileAvatar`** - Identity avatar for account, member, or user profile surfaces

**`PresenceAvatar`** - Presence-aware avatar with online status metadata

**`TeamAvatarGroup`** - Grouped avatars for team, roster, or participant clusters

### Molecules

**`AvatarBadge`** - Avatar molecule with the online presence indicator

**`AvatarGroup`** - Medium stacked avatar molecule with optional overflow counter

### Atom

**`Avatar`** - Base avatar atom for initials, icon fallback, and image content
