# Button Components

Marwes provides a powerful button system with two complementary approaches:
**semantic components** for common patterns and **raw props** for advanced control.

## Recommended: Semantic Components

Use purpose-built components that automatically set AI-friendly metadata:

```tsx
import { CancelButton, SubmitButton, CreateButton, DangerButton, LinkButton } from "@marwes-ui/react"

// ✅ Preferred - auto-sets action="cancel", variant="secondary"
<CancelButton onClick={handleCancel}>Cancel</CancelButton>

// ✅ Preferred - auto-sets action="submit", uses form submission
<SubmitButton>Save Changes</SubmitButton>

// ✅ Preferred - auto-sets action="create"
<CreateButton onClick={handleCreate}>New Project</CreateButton>

// ✅ Preferred - auto-sets action="delete", data-destructive="true"
<DangerButton onClick={handleDelete}>Delete</DangerButton>

// ✅ Preferred - auto-sets action="navigate", renders as anchor
<LinkButton href="/dashboard">Go to Dashboard</LinkButton>
```

### Why Semantic Components?

1. **Guaranteed AI context** - action metadata is always set correctly
2. **Consistent UX** - enforces design patterns across your app
3. **Less boilerplate** - no need to remember which props to set
4. **Self-documenting** - code intent is clear from component name

## Advanced: Raw Props

For dynamic or custom scenarios, use the base `Button` with explicit props:

```tsx
import { Button, ButtonVariant, ButtonAction, ButtonSize } from "@marwes-ui/react"

// Dynamic action based on state
<Button
  variant={ButtonVariant.primary}
  size={ButtonSize.md}
  action={isEditing ? ButtonAction.edit : ButtonAction.create}
>
  {isEditing ? "Update" : "Create"}
</Button>

// Custom action not covered by semantic components
<Button
  variant={ButtonVariant.secondary}
  action={ButtonAction.reset}
>
  Reset Form
</Button>
```

## Variant vs Action

### `variant` — Visual Style

Controls how the button **looks**. Choose based on visual hierarchy:

- **`primary`** — Filled, high emphasis (main actions)
- **`secondary`** — Outlined, medium emphasis (secondary actions)  
- **`text`** — Minimal, low emphasis (tertiary/inline actions)

### `action` — Semantic Purpose

Controls what the button **does** (AI-readable metadata):

- **`submit`** — Form submission
- **`cancel`** — Dismiss/cancel operation
- **`create`** — Create new item
- **`delete`** — Remove/destroy item
- **`navigate`** — Go to another page
- **`edit`** — Modify existing item
- **`reset`** — Reset form state
- **`button`** — Generic action (default)

> **Note:** These are independent. A `DangerButton` uses `variant="primary"` (filled) with error styling, while `CancelButton` uses `variant="secondary"` (outlined).

## Component Reference

### General Buttons

**`PrimaryButton`** — General primary actions  
Sets: `variant="primary"`

**`SecondaryButton`** — General secondary actions  
Sets: `variant="secondary"`

**`TextButton`** — Subtle/inline actions  
Sets: `variant="text"`

### Purpose Buttons (AI-Friendly)

**`SubmitButton`** — Form submissions  
Sets: `action="submit"`

**`CancelButton`** — Cancel/dismiss actions  
Sets: `action="cancel"`, `variant="secondary"`

**`CreateButton`** — Creating new items  
Sets: `action="create"`

**`DangerButton`** — Destructive actions  
Sets: `action="delete"`, error styling, `data-destructive="true"`

**`LinkButton`** — Navigation links  
Sets: `action="navigate"`, renders as `<a>` element
