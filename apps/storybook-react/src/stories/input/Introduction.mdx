# Input Components

Marwes provides purpose-specific field components that encapsulate best practices
and automatically set AI-friendly metadata. Each field variant is optimized for
its specific use case with appropriate semantic HTML attributes.

Input domain groups everything related to text-like data entry in one place,
using a clear Atom -> Molecule -> Purpose progression.

### Field naming

Components that wrap raw input-domain atoms with label, helper text, and error semantics
are called **Fields** (e.g. `InputField`, `SelectField`, `TextareaField`, `RichTextField`, `DropdownField`, `SearchField`, `EmailField`).
The suffix signals that the component provides full field-level UX — accessible labelling,
validation display, and AI-friendly `data-purpose` attributes — so you don't have to wire
those concerns yourself.

## Atom: Input

Use `Input` directly when you need full layout control without label/helper/error wrappers:

```tsx
import { Input } from "@marwes-ui/react"

<Input
  type="text"
  placeholder="Type here"
  value={value}
  onValueChange={setValue}
/>
```

## Atom: Select

Use `Select` for the compact select atom. Set `native={true}` when you explicitly want the browser's built-in dropdown chrome:

```tsx
import { Select } from "@marwes-ui/react"

<Select
  placeholder="Choose a country"
  options={[
    { value: "se", label: "Sweden" },
    { value: "us", label: "United States" },
  ]}
  value={country}
  onValueChange={setCountry}
/>
```

## Atom: Textarea

Use `Textarea` when you need multiline freeform text entry without label/helper/error wrappers:

```tsx
import { Textarea } from "@marwes-ui/react"

<Textarea
  rows={4}
  placeholder="Add more detail"
  value={value}
  onValueChange={setValue}
/>
```

## Atom: RichText

Use `RichText` when the authored content itself needs inline emphasis such as bold, italic, or underline. Use `Textarea` when the content should stay plain text.

```tsx
import { RichText } from "@marwes-ui/react"

<RichText
  placeholder="Write a formatted update"
  value={html}
  onValueChange={setHtml}
/>
```

## Recommended: Purpose Input Components

Use purpose-built input components that automatically set the correct `type`, `inputMode`, `autoComplete`, and other semantic attributes:

```tsx

  DropdownField,
  SearchField,
  PasswordField,
  EmailField,
  DateOfBirthField,
  ZipCodeField,
  PhoneField,
  URLField,
  CurrencyField

// ✅ Figma dropdown presentation with locked Marwes styling
<DropdownField

  select={{
    placeholder: "Choose a country",
    options: [
      { value: "se", label: "Sweden" },
      { value: "us", label: "United States" },

// ✅ Search with automatic clear button
<SearchField
  label="Search products"

// ✅ Date of birth with native date semantics and birthday autofill
```

## When to Use Each Field

### DropdownField

- Figma-aligned dropdown presentation
- Cases where the custom Marwes open-state should stay visually consistent
- Forms that need the same field semantics as `SelectField`, but a fixed visual treatment

**Example:**

```tsx
<DropdownField
  label="Country"
  helperText="Uses the Marwes dropdown presentation."
  select={{
    value: country,
    onValueChange: setCountry,
    options: [
      { value: "se", label: "Sweden" },
      { value: "us", label: "United States" },
    ],
  }}
/>
```

### SearchField

- Search bars and filters
- Product search
- User/content lookup
- Any search functionality

**Example:**

```tsx
<SearchField
  label="Search orders"
  helperText="Search by order number, customer name, or product"
  input={{ value: search, onValueChange: setSearch }}
/>
```

### PasswordField

- Login/signup forms
- Password change forms
- Security settings

**Props:**

- `autoComplete="current-password"` (default) - For login
- `autoComplete="new-password"` - For signup/reset

**Example:**

```tsx
<PasswordField
  label="New Password"
  autoComplete="new-password"
  helperText="Must be at least 8 characters"
  input={{ value: password, onValueChange: setPassword }}
/>
```

### EmailField

- Email address inputs
- Contact forms
- Newsletter signups
- User registration

**Example:**

```tsx
<EmailField
  label="Email address"
  helperText="We'll send a confirmation to this email"
  input={{ value: email, onValueChange: setEmail }}
/>
```

### DateOfBirthField

- Birthday collection in signup or onboarding forms
- Age-gated flows that need a real calendar date
- Profile details that benefit from `autoComplete="bday"`

**Example:**

```tsx
<DateOfBirthField
  label="Date of birth"
  helperText="Used for age verification and onboarding."
  input={{ value: dateOfBirth, onValueChange: setDateOfBirth }}
/>
```

### ZipCodeField

- Shipping, billing, and service-availability forms
- Postal-code capture that should preserve leading zeros
- Address flows that benefit from `autoComplete="postal-code"`

**Example:**

```tsx
<ZipCodeField
  label="ZIP code"
  helperText="Used for shipping estimates and tax lookup."
  input={{ value: zipCode, onValueChange: setZipCode }}
/>
```

### PhoneField

- Phone number inputs
- Contact information
- SMS verification setup

**Note:** Formatting should be handled by parent component or library.

**Example:**

```tsx
<PhoneField
  label="Phone number"
  helperText="Include country code for international numbers"
  input={{ value: phone, onValueChange: setPhone }}
/>
```

### URLField

- Website/portfolio links
- Social media profiles
- Resource links
- Any URL input

**Example:**

```tsx
<URLField
  label="Portfolio website"
  helperText="Include https:// or http://"
  input={{ value: url, onValueChange: setUrl }}
/>
```

### CurrencyField

- Prices and monetary amounts
- Payment forms
- Financial data entry
- Donation amounts

**Props:**

- `currency` - ISO currency code (USD, EUR, GBP, etc.)

**Note:** Uses `type="text"` for better control. Formatting should be handled by parent.

**Controlled usage recommended:** Pass both `input.value` and `input.onValueChange` when you want the sanitized currency value to stay in sync with what the user sees.

**Example:**

```tsx
<CurrencyField
  label="Product price"
  currency="USD"
  input={{ value: price, onValueChange: setPrice }}
/>
```

## Molecule: InputField

Use the base `InputField` when you want built-in label, helper text, and error wiring:

```tsx
import { InputField } from "@marwes-ui/react";

<InputField
  label="Custom field"
  input={{
    type: "text",
    inputMode: "numeric",
    pattern: "[0-9]*",
    value: value,
    onValueChange: setValue,
  }}
/>;
```

## Molecule: SelectField

Use `SelectField` when a select control also needs label, helper text, and error wiring:

```tsx
import { SelectField } from "@marwes-ui/react";

<SelectField
  label="Country"
  helperText="Used to tailor currency and shipping defaults."
  select={{
    placeholder: "Choose a country",
    options: [
      { value: "se", label: "Sweden" },
      { value: "us", label: "United States" },
    ],
    value: country,
    onValueChange: setCountry,
  }}
/>
```

## Molecule: TextareaField

Use `TextareaField` for multiline content that still needs the same label, helper text, and error treatment:

```tsx
import { TextareaField } from "@marwes-ui/react";

<TextareaField
  label="Project description"
  helperText="A short summary is usually enough."
  textarea={{
    rows: 4,
    value: description,
    onValueChange: setDescription,
  }}
/>
```

## Molecule: RichTextField

Use `RichTextField` when the editor needs the same label, helper text, and error treatment as other Marwes field wrappers:

> **Manual review note**
> `RichTextField` inherits the same manual-review boundary as `RichText`.
> Treat automated tests as contract protection, not a full guarantee of editor accessibility in every browser and assistive technology pairing.

```tsx
import { RichTextField } from "@marwes-ui/react";

<RichTextField
  label="Release notes"
  helperText="Use bold and italics to call out important changes."
  editor={{
    value: notes,
    onValueChange: setNotes,
    placeholder: "Write a formatted summary",
  }}
/>
```

## Common Props

Input-based field components support the same base props:

```tsx
type BaseFieldProps = {
  label: string; // Required: Field label
  helperText?: string; // Optional: Helper text below input
  error?: string; // Optional: Error message (marks invalid)
  id?: string; // Optional: Custom ID (auto-generated if omitted)
  ariaDescribedBy?: string; // Optional: Additional aria-describedby IDs
  input: {
    value?: string; // Controlled value
    defaultValue?: string; // Uncontrolled default value
    onValueChange?: (value: string) => void;
    placeholder?: string;
    disabled?: boolean;
    readOnly?: boolean;
    required?: boolean;
    // ... and other input-specific props
  };
};
```

`SelectField` uses the same outer props (`label`, `helperText`, `error`, `id`, `ariaDescribedBy`) but forwards select control props under `select`. Native browser select behavior is the default. Use `select.native={false}` to opt into the custom Marwes dropdown state from Figma node `1364:7701`. `TextareaField` does the same for multiline control props under `textarea`.

`DropdownField` reuses the `SelectField` API, defaults `select.native` to `false`, and acts as the purpose-level wrapper for the custom Marwes dropdown presentation. Pass `select.native={true}` only when you intentionally want browser chrome instead.

## Controlled vs Uncontrolled

### Controlled (Recommended)

```tsx
const [email, setEmail] = useState("")

<EmailField
  label="Email"
  input={{
    value: email,
    onValueChange: setEmail
  }}
/>
```

### Uncontrolled

```tsx
<EmailField
  label="Email"
  input={{
    defaultValue: "user@example.com",
  }}
/>
```

## Validation Pattern

```tsx
const [email, setEmail] = useState("");
const [error, setError] = useState<string>();

const validateEmail = (value: string) => {
  if (!value) {
    setError("Email is required");
  } else if (!/^[^\s@]+@[^\s@]+\.[^\s@]+$/.test(value)) {
    setError("Please enter a valid email address");
  } else {
    setError(undefined);
  }
};

<EmailField
  label="Email address"
  error={error}
  input={{
    value: email,
    onValueChange: (value) => {
      setEmail(value);
      validateEmail(value);
    },
  }}
/>;
```

## AI-Friendly Attributes

All field variants automatically set `data-purpose` attributes for AI parsing:

```html
<div data-purpose="email">
  <label>Email address</label>
  <input type="email" inputmode="email" autocomplete="email" />
</div>

<div data-purpose="password">
  <label>Password</label>
  <input type="password" autocomplete="current-password" />
  <button aria-label="Show password">👁</button>
</div>

<div data-purpose="currency" data-currency="USD">
  <label>Amount</label>
  <input type="text" inputmode="decimal" />
  <p>Amount in USD</p>
</div>
```

These attributes help AI tools understand field purpose and provide better assistance.
