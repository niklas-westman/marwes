# Dialog Components

Marwes provides dialogs at three layers: **purpose components** for common modal
workflows, **DialogModal** for stateful overlay composition, and the raw
**Dialog** atom for custom shells.

## Recommended: Purpose Components

Use purpose-built components when the dialog intent is already known:

```tsx
import { ConfirmDialog, DestructiveDialog, InfoDialog, Paragraph } from "@marwes-ui/react"

<ConfirmDialog open title="Publish update" description="This sends the release immediately.">
  <Paragraph>Review the release summary before confirming.</Paragraph>
</ConfirmDialog>

<DestructiveDialog open title="Delete workspace" description="This action cannot be undone.">
  <Paragraph>Export anything you need before continuing.</Paragraph>
</DestructiveDialog>

<InfoDialog open title="Scheduled maintenance" description="The workspace will be read-only briefly.">
  <Paragraph>Save drafts before the maintenance window begins.</Paragraph>
</InfoDialog>
```

## Base: DialogModal (Molecule)

Use `DialogModal` when you need custom content or action wiring, but still want
the library to manage the modal overlay.

```tsx
import {
  CancelButton,
  ConfirmButton,
  DialogModal,
  Paragraph,
} from "@marwes-ui/react"

<DialogModal
  open
  title="Invite teammates"
  description="Select a role and invite collaborators to the workspace."
  footer={({ close }) => (
    <>
      <CancelButton onClick={close}>Cancel</CancelButton>
      <ConfirmButton onClick={close}>Continue</ConfirmButton>
    </>
  )}
>
  <Paragraph>Choose a role before sending the invitation.</Paragraph>
</DialogModal>
```

### DialogModal Features

- **Portal + scrim** - renders above the app surface with an internal backdrop
- **Controlled or uncontrolled state** - use `open` or `defaultOpen`
- **Dismiss behavior** - supports close button, Escape, and scrim click rules
- **Focus restoration** - returns focus to the prior trigger when the modal closes

## Low-level: Dialog (Atom)

Use the raw `Dialog` atom when parent code already owns the overlay and state
management, or when you need a custom shell composition.

```tsx
import { CancelButton, ConfirmButton, Dialog, Paragraph } from "@marwes-ui/react"

<Dialog
  title="Dialog title"
  description="Describe the purpose of the dialog."
  footer={
    <>
      <CancelButton>Cancel</CancelButton>
      <ConfirmButton>Confirm</ConfirmButton>
    </>
  }
>
  <Paragraph>Custom content goes here.</Paragraph>
</Dialog>
```

## Component Reference

### Purpose Components

**`ConfirmDialog`** - Neutral confirmation and approval prompts

**`DestructiveDialog`** - Irreversible or high-risk actions

**`InfoDialog`** - Read-and-acknowledge messaging

### Base Molecule

**`DialogModal`** - Modal overlay with portal, scrim, dismissal, and focus handling

### Atom

**`Dialog`** - Low-level shell with header, content, and optional footer

- `Dialog (Atom)`
- `DialogModal`
- `ConfirmDialog`
- `DestructiveDialog`
- `InfoDialog`
