# Card Components

Marwes provides cards at two layers: **purpose components** for common card
patterns and the base **Card** atom for custom composition.

## Component Architecture

```
Card (Atom)
     ↓
Purpose Wrappers
  ├─ ProductCard
  ├─ ProfileCard
  └─ StatCard
```

## Recommended: Purpose Components

Use purpose-built cards when the card has a clear product meaning:

```svelte
<script>
  import { ProductCard, ProfileCard, StatCard } from "@marwes-ui/svelte"
</script>

<ProductCard title="Pro plan">
  Unlimited projects, advanced analytics, and priority support.
</ProductCard>

<ProfileCard title="Alex Morgan">
  Design systems engineer with ownership of component primitives.
</ProfileCard>

<StatCard title="Monthly recurring revenue">
  $128,400
</StatCard>
```

## Atom: Card

Use the raw `Card` atom for any content grouping:

```svelte
<script>
  import { Card } from "@marwes-ui/svelte"
</script>

<Card title="Getting started">
  Install via pnpm and wrap your app with MarwesProvider.
</Card>

<Card>
  Body-only card — no title.
</Card>
```

## Usage Notes

- `title` renders an optional heading inside the card
- Children are rendered as the card body
- Card provides consistent padding, border radius, and elevation
- Structure: `mw-card` → `mw-card__header` + `mw-card__title` → `mw-card__body`

## Component Reference

### Purpose Components

**`ProductCard`** — Product pricing and feature cards (`data-purpose="product-card"`)

**`ProfileCard`** — User or team member profiles (`data-purpose="profile-card"`)

**`StatCard`** — Key metrics and KPI cards (`data-purpose="stat-card"`)

### Atom

**`Card`** — Container atom for grouping related content
