# Heading Components

Marwes provides `H1`, `H2`, and `H3` as atom-level typography primitives for
semantic structure and visual hierarchy.

## Component Architecture

```
H1 / H2 / H3 (Atoms)
```

## Recommended Usage

Choose the heading level by document outline first, then use `size` only when the
visual scale needs to differ from the semantic level:

```svelte
<script>
  import { H1, H2, H3 } from "@marwes-ui/svelte"
</script>

<H1>Page Title</H1>
<H2>Section Heading</H2>
<H3>Subsection Heading</H3>

<!-- Visual override: semantic h1, visual h2 -->
<H1 size="h2">Visually smaller but semantically h1</H1>
```

## Usage Notes

- Prefer `H1` for the page-level title, then nest `H2` and `H3` by content structure
- Use `size` when semantic level and visual scale need to differ
- Add `id` when headings should support anchor links or table-of-contents jumps

## Accessibility Notes

**Choose the heading level by document outline first.** Screen readers use heading levels
to navigate page structure. Pick the level that matches content hierarchy, not font size.

**`size` changes the visual scale, not the semantic level.** An `H3` with `size="h1"`
is still an `<h3>` for assistive technology.

**Do not use headings for decorative emphasis.** If text needs to look larger but doesn't
introduce a real section, use another typography surface.

## Component Reference

**`H1`** — Top-level heading atom for page titles

**`H2`** — Section heading atom for major content groupings

**`H3`** — Subsection heading atom for nested content
