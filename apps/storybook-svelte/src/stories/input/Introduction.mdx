# Input Components

Marwes provides input-domain components organized in a clear Atom → Molecule → Purpose
progression for text-based data entry.

## Input Architecture

```
Atoms
  ├─ Input
  ├─ Select
  └─ Textarea

Molecules
  ├─ InputField
  ├─ SelectField
  └─ TextareaField

Purpose Fields (Input-based)
  ├─ PasswordField
  ├─ EmailField
  ├─ SearchField
  ├─ PhoneField
  ├─ URLField
  ├─ CurrencyField
  ├─ DateOfBirthField
  ├─ ZipCodeField
  └─ DropdownField
```

## Recommended: Purpose Fields

Use purpose-built components for common input patterns:

```svelte
<script>
  import { PasswordField, EmailField, SearchField, CurrencyField } from "@marwes-ui/svelte"
</script>

<PasswordField label="Password" input={{ placeholder: "Enter password" }} />
<EmailField label="Email" input={{ placeholder: "you@example.com" }} />
<SearchField label="Search" input={{ placeholder: "Search..." }} />
<CurrencyField label="Amount" input={{ placeholder: "0.00" }} />
```

## Base: InputField (Molecule)

The recommended molecule for most form inputs:

```svelte
<script>
  import { InputField } from "@marwes-ui/svelte"
  let name = $state("")
</script>

<InputField label="Full name" input={{ placeholder: "Jane Doe" }} bind:value={name} />
<InputField label="Username" helperText="3-20 characters." input={{ placeholder: "johnsmith" }} />
<InputField label="Password" error="Must be 8+ characters." input={{ type: "password" }} />
<InputField label="Amount" leadingSymbol="$" input={{ placeholder: "0.00" }} />
```

### InputField Features

- **Visible label** wired via `for`/`id`
- **Helper text** and **error regions** with `aria-describedby` wiring
- **Leading symbol** for currency/prefix display
- **Password toggle** — eye icon shows/hides password text
- **Search affordances** — search icon when empty, clear button when filled
- **`bind:value`** for two-way data binding

## Base: SelectField (Molecule)

```svelte
<script>
  import { SelectField } from "@marwes-ui/svelte"
</script>

<SelectField
  label="Country"
  select={{ options: [
    { value: "se", label: "Sweden" },
    { value: "us", label: "United States" },
  ], placeholder: "Choose a country" }}
/>
```

## Base: TextareaField (Molecule)

```svelte
<script>
  import { TextareaField } from "@marwes-ui/svelte"
</script>

<TextareaField label="Message" textarea={{ placeholder: "Enter your message...", rows: 4 }} />
```

## Low-level: Atoms

Use atoms when you need full control:

```svelte
<script>
  import { Input, Select, Textarea } from "@marwes-ui/svelte"
</script>

<Input placeholder="Raw input" />
<Select options={[{ value: "a", label: "A" }]} />
<Textarea placeholder="Raw textarea" />
```

### Select Native vs Custom

Select defaults to the Marwes visual treatment (custom chevron icon). Use `native={true}` for browser dropdown chrome:

```svelte
<Select options={options} native />
```

## Component Reference

### Purpose Fields

**`PasswordField`** — Password input with visibility toggle (`type="password"`)

**`EmailField`** — Email input with email keyboard (`type="email"`)

**`SearchField`** — Search input with icon and clear button (`type="search"`)

**`PhoneField`** — Phone input with tel keyboard (`type="tel"`)

**`URLField`** — URL input (`type="url"`)

**`CurrencyField`** — Currency input with leading symbol and decimal keyboard

**`DateOfBirthField`** — Date input with native picker (`type="date"`, `autoComplete="bday"`)

**`ZipCodeField`** — Postal code with numeric keyboard (`inputMode="numeric"`)

**`DropdownField`** — SelectField with dropdown presentation (`data-purpose="dropdown"`)

### Molecules

**`InputField`** — Accessible text input with label, helper, error, password toggle, search affordances

**`SelectField`** — Accessible select with label, helper, error

**`TextareaField`** — Accessible textarea with label, helper, error

### Atoms

**`Input`** — Low-level `<input>` with recipe styling

**`Select`** — Native or custom-styled `<select>`

**`Textarea`** — Low-level `<textarea>` with recipe styling
