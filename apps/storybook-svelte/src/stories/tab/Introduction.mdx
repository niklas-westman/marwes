# Tab Components

Marwes provides tabs at three layers: **purpose components** for common tab sets,
**TabGroup** for accessible tablist and panel coordination, and purpose wrappers
for semantic context.

## Recommended: Purpose Components

Use purpose-built components when the tab set has a clear product meaning:

```svelte
<script>
  import { NavigationTabs, ContentTabs, SettingsTabs } from "@marwes-ui/svelte"
</script>

<NavigationTabs
  tabs={[
    { value: "overview", label: "Overview", panel: "Key product summary." },
    { value: "roadmap", label: "Roadmap", panel: "Upcoming milestones." },
  ]}
/>

<ContentTabs
  tabs={[
    { value: "summary", label: "Summary", panel: "Project overview." },
    { value: "results", label: "Results", panel: "Performance gains." },
  ]}
/>

<SettingsTabs
  tabs={[
    { value: "general", label: "General", panel: "Defaults and workspace info." },
    { value: "security", label: "Security", panel: "Access policies and SSO." },
  ]}
/>
```

## Base: TabGroup (Molecule)

Use `TabGroup` when you need custom tab labels or panels:

```svelte
<script>
  import { TabGroup } from "@marwes-ui/svelte"
</script>

<TabGroup
  ariaLabel="Profile sections"
  tabs={[
    { value: "overview", label: "Overview", panel: "Profile summary." },
    { value: "activity", label: "Activity", panel: "Recent actions." },
    { value: "security", label: "Security", panel: "Access and sessions." },
  ]}
  defaultActiveTab="overview"
/>
```

### TabGroup Features

- **Keyboard navigation built in** — Left/Right arrows plus Home/End move between enabled tabs
- **Controlled or uncontrolled state** — use `activeTab` for external state or `defaultActiveTab` for internal state
- **Accessible panel wiring** — each tab connects to its `tabpanel` automatically
- **Disabled tabs** — individual tabs can be disabled via the `disabled` property

## Component Reference

### Purpose Components

**`NavigationTabs`** — Peer-view navigation and section switching (`data-purpose="navigation-tabs"`)

**`ContentTabs`** — Segmented content surfaces and reading flows (`data-purpose="content-tabs"`)

**`SettingsTabs`** — Configuration areas and admin groupings (`data-purpose="settings-tabs"`)

### Base Molecule

**`TabGroup`** — Accessible tablist plus panel coordination
